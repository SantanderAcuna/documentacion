Volver al Inicio

1. Crear Llave SSH Personalizada

Generar clave SSH ed25519

Este comando genera una clave SSH utilizando el algoritmo ed25519, que es más seguro y eficiente que RSA. La opción -t especifica el tipo de algoritmo y -f especifica el nombre del archivo donde se guardará la clave.

ssh-keygen -t ed25519 -f ~/.ssh/vps_portal
Nota importante: Durante la generación de la clave, se te preguntará si deseas establecer una frase de contraseña (passphrase). Es altamente recomendable establecer una frase de contraseña para mayor seguridad, aunque esto significa que tendrás que ingresarla cada vez que uses la clave.

Verificar la creación de la llave SSH

Este comando lista los archivos en el directorio ~/.ssh/ para verificar que los archivos de clave se hayan creado correctamente.

ls ~/.ssh

Deberías ver los siguientes archivos:

  • vps_portal - Clave privada (NUNCA compartir este archivo)
  • vps_portal.pub - Clave pública (este archivo se coloca en el servidor)

Mostrar clave pública

Este comando muestra el contenido de la clave pública que necesitarás copiar y agregar al archivo ~/.ssh/authorized_keys en el servidor VPS.

cat ~/.ssh/vps_portal.pub
Advertencia de seguridad: La clave privada (vps_portal) debe permanecer en tu máquina local y nunca debe ser compartida, enviada por correo electrónico o subida a ningún servicio en la nube. Solo la clave pública (vps_portal.pub) debe ser colocada en el servidor.

Configurar para que se lea la clave creada para SSH

Este comando prueba la conexión SSH utilizando la clave específica que acabamos de crear. La opción -i especifica la ruta a la clave privada.

ssh -i ~/.ssh/vps_portal root@45.55.213.131

Si es la primera vez que te conectas a este servidor, verás una advertencia sobre la autenticidad del host. Debes verificar que la huella digital (fingerprint) coincida con la del servidor y luego escribir "yes" para continuar.

Ingresar al VPS (comando definitivo)

Una vez que la clave pública esté correctamente configurada en el servidor, este comando te permitirá acceder al VPS sin necesidad de contraseña, utilizando solo la clave SSH.

ssh -i ~/.ssh/vps_portal root@45.55.213.131

2. Configuración SSH para Conexión Simplificada

Crear archivo de configuración SSH

Se debe crear un archivo llamado config (sin extensión) en el directorio SSH del usuario.

# Configuración del tiempo de conexión ServerAliveInterval 120 ServerAliveCountMax 3 # Alias para el VPS principal Host portal-web HostName 45.55.213.131 User root IdentityFile "/c/Users/Jose acuña/.ssh/vps_portal" # Alias para usuario alternativo Host joseacuna HostName 45.55.213.131 User sacunapolo IdentityFile "/c/Users/Jose acuña/.ssh/sacunapolo"

3. Acceso al VPS

Acceso con alias configurado

ssh portal-web

Después de configurar el archivo ~/.ssh/config, puedes usar este comando simplificado.

systemctl status unattended-upgrades

systemctl restart unattended-upgrades

Reiniciar el servidor

7. Conectar el VPS a GitHub

Generar una nueva clave SSH en el VPS

ssh-keygen

Iniciar el agente SSH

ssh-agent -s

Agregar la clave SSH al agente

ssh-add ~/.ssh/id_ed25519

Mostrar la clave pública

cat id_ed25519.pub

Probar la conexión a GitHub

ssh -T git@github.com

8. Configuración de Cron Jobs

Crear un archivo de prueba

date >> /root/time.txt

Editar la tabla de cron

crontab -e

9. Asignar Cuentas de Usuario al VPS

Crear un nuevo usuario

adduser joseacuna

Eliminar un usuario

deluser test --remove-home

10. Asignando Permisos de Administrador

Agregar usuario al grupo sudo

sudo usermod -aG sudo sacunapolo

11. Seguridad SSH

Editar configuración SSH

sudo nano /etc/ssh/sshd_config

Cambiar:

PermitRootLogin no PasswordAuthentication no

Refrescar la configuración SSH

sudo systemctl reload ssh.service

12. Configurar Firewall (UFW)

Verificar estado del firewall

sudo ufw status

Permitir tráfico SSH

sudo ufw allow OpenSSH

Habilitar el firewall

sudo ufw enable

13. Permisos de Carpeta

Cambiar propietario de una carpeta

sudo chown -R sacunapolo test/

14. Instalar Fail2Ban

Instalar Fail2Ban

sudo apt install fail2ban

Crear archivo de configuración

[DEFAULT] bantime = 3h maxretry = 3 [sshd] enabled = true

Verificar estado de Fail2Ban

sudo fail2ban-client status

15. Configuración de Nginx

Instalar Nginx

sudo apt update sudo apt install nginx

Permitir tráfico web en el firewall

sudo ufw allow 'Nginx Full'

Crear configuración para el sitio web

server { listen 80 default_server; listen [::]:80 default_server; root /var/www/portalweb; server_name labsoftware.xyz; index index.html; location / { try_files $uri $uri/ =404; } }

16. Instalar MySQL

Instalar MySQL Server

sudo apt update sudo apt install mysql-server

Configurar seguridad de MySQL

sudo mysql_secure_installation

Crear base de datos y usuario

CREATE DATABASE portalweb; CREATE USER 'sacunapolo'@'localhost' IDENTIFIED WITH mysql_native_password BY 's41uD+v1da'; GRANT ALL PRIVILEGES ON portalweb.* TO 'sacunapolo'@'localhost'; FLUSH PRIVILEGES;

17. Túneles SSH para MySQL

Crear túnel SSH para MySQL

ssh sacunapolo -L 3333:localhost:3306 -N

18. Seguridad Nginx

Proteger archivos sensibles

location ~ /\.ht { deny all; } location ~ /\.git { deny all; }

Ocultar versión de Nginx

server_tokens off;

20. Protección DDoS

Configuración de protección DDoS

limit_conn_zone $binary_remote_addr zone=conn_limit_per_ip:15m; limit_req_zone $binary_remote_addr zone=req_limit_per_ip:15m rate=40r/s; limit_conn conn_limit_per_ip 40; limit_req zone=req_limit_per_ip burst=40 nodelay; client_body_timeout 5s; client_header_timeout 5s; send_timeout 5; client_body_buffer_size 256k; client_header_buffer_size 2k; client_max_body_size 3m; large_client_header_buffers 2 2k;

21. Evitar Secuestro Nginx

Configurar redirección para servidor por defecto

server { listen 80 default_server; listen [::]:80 default_server; server_name _; return 301 http://labsoftware.xyz; }

22. Let's Encrypt

Instalar Certbot

sudo apt update sudo snap install --classic certbot

Configurar Certbot para Nginx

sudo certbot --nginx --hsts -d labsoftware.xyz -d www.labsoftware.xyz

Configurar HTTP/2

listen [::]:443 ssl ipv6only=on http2; # managed by Certbot listen 443 ssl http2; # managed by Certbot ssl_certificate /etc/letsencrypt/live/labsoftware.xyz/fullchain.pem; ssl_certificate_key /etc/letsencrypt/live/labsoftware.xyz/privkey.pem; include /etc/letsencrypt/options-ssl-nginx.conf; ssl_dhparam /etc/letsencrypt/ssl-dhparams.pem; add_header Strict-Transport-Security "max-age=31536000" always;

24. Transferencia de Archivos con SCP

Importante: Todos los comandos SCP deben ejecutarse desde Git Bash en Windows. No usar CMD ni PowerShell. Git Bash proporciona el entorno Unix necesario.
PARTE 1: Copiar del PC LOCAL → VPS

Paso 1: Abrir Git Bash

Para transferir archivos, necesitas usar Git Bash en Windows:

  • Menú Inicio → Buscar "Git Bash" y abrirlo
  • O hacer clic derecho en cualquier carpeta → "Git Bash Here"
Nota: No uses CMD, PowerShell o la terminal integrada de VS Code. Solo Git Bash funciona correctamente.

Paso 2: Ir a la carpeta local en Git Bash

Navegar a la carpeta donde tienes los archivos que quieres transferir:

cd /c/laragon/www/docuemntacion/documentacion ls -la

Esto te mostrará todos los archivos en esa carpeta, por ejemplo:

  • index.html
  • documentacion.html
  • README.md

Paso 3: Sintaxis básica de SCP (PC → VPS)

La estructura básica del comando es:

scp [opciones] archivo_local usuario@servidor:ruta_remota

Opciones más usadas:

Opción Descripción
-r Copia carpetas completas (recursivo)
-P 2222 Especifica puerto SSH diferente (no 22)
-v Modo verboso (muestra detalles)
-i clave.pem Usa una clave SSH específica

Paso 4: Copiar archivos individuales (PC → VPS)

4.1 Copiar un solo archivo:

scp index.html sacunapolo:/home/sacunapolo/

4.2 Copiar varios archivos específicos:

scp index.html documentacion.html sacunapolo:/home/sacunapolo/

4.3 Copiar todos los archivos HTML:

scp *.html sacunapolo:/home/sacunapolo/

4.4 Copiar y cambiar nombre en destino:

scp index.html sacunapolo:/home/sacunapolo/index-backup.html

Paso 5: Copiar carpetas completas (PC → VPS)

5.1 Copiar carpeta completa con todo su contenido:

cd /c/laragon/www/docuemntacion scp -r documentacion sacunapolo:/home/sacunapolo/

5.2 Copiar solo el contenido de la carpeta (sin la carpeta misma):

cd /c/laragon/www/docuemntacion/documentacion scp -r * sacunapolo:/home/sacunapolo/

Paso 6: Verificar que los archivos llegaron al VPS

ssh sacunapolo cd /home/sacunapolo/ ls -la exit

Esto te conectará al VPS, listará los archivos en tu carpeta home, y luego cerrará la conexión.

Paso 7: Mover archivos a la ruta web (si es necesario)

Si necesitas mover los archivos al directorio web de Nginx:

ssh sacunapolo sudo mv /home/sacunapolo/*.html /var/www/desarrollo.labsoftware.xyz/ exit
Consejo: Puedes usar scp directamente a la carpeta web si tienes permisos:
scp index.html sacunapolo:/var/www/desarrollo.labsoftware.xyz/
PARTE 2: Copiar del VPS → PC LOCAL

Paso 1: Ir a la carpeta destino en tu PC

Abrir Git Bash y navegar a donde quieres guardar los archivos:

cd /c/Users/"Jose acuña"/Desktop

O si quieres crear una carpeta nueva:

mkdir backup_vps cd backup_vps

Paso 2: Sintaxis básica de SCP (VPS → PC)

La estructura es similar pero invertida:

scp usuario@servidor:ruta_remota ruta_local

El punto . significa "carpeta actual".

Paso 3: Copiar archivos individuales (VPS → PC)

3.1 Copiar un archivo específico:

scp sacunapolo:/home/sacunapolo/index.html .

3.2 Copiar varios archivos por patrón:

scp sacunapolo:/home/sacunapolo/*.html .

3.3 Copiar desde el directorio web del servidor:

scp sacunapolo:/var/www/desarrollo.labsoftware.xyz/index.html .

3.4 Copiar a una ruta específica:

scp sacunapolo:/home/sacunapolo/config.txt /c/laragon/www/backups/

Paso 4: Copiar carpetas completas (VPS → PC)

4.1 Copiar carpeta completa con todo su contenido:

scp -r sacunapolo:/home/sacunapolo/documentacion /c/laragon/www/backups/

4.2 Copiar el contenido a una carpeta local:

mkdir backup_servidor scp -r sacunapolo:/home/sacunapolo/* backup_servidor/

4.3 Copiar configuración de Nginx:

scp -r sacunapolo:/etc/nginx/sites-available /c/backups/nginx/

Paso 5: Verificar la copia local

ls -la du -sh *

El primer comando lista los archivos, el segundo muestra el tamaño de cada uno.

Resumen Rápido de Comandos SCP
Acción Comando Ejemplo
PC → VPS (archivo) scp archivo usuario@host:ruta scp index.html sacunapolo:/home/
VPS → PC (archivo) scp usuario@host:archivo . scp sacunapolo:/home/index.html .
PC → VPS (carpeta) scp -r carpeta usuario@host:ruta scp -r docs sacunapolo:/var/www/
VPS → PC (carpeta) scp -r usuario@host:carpeta . scp -r sacunapolo:/var/www/html .
Puerto diferente scp -P puerto ... scp -P 2222 index.html sacunapolo:/home/
Con clave SSH scp -i clave ... scp -i ~/.ssh/vps_portal index.html sacunapolo:/home/
Solución de Problemas Comunes

❌ Error: "No such file or directory"

Solución: Verifica que el archivo existe y la ruta es correcta:

ls -la archivo.html scp ./archivo.html sacunapolo:/home/sacunapolo/

❌ Error: "Permission denied"

Solución 1: Copiar a /tmp y luego mover con sudo:

scp archivo.html sacunapolo:/tmp/ ssh sacunapolo sudo mv /tmp/archivo.html /var/www/html/

Solución 2: Usar usuario con permisos:

scp archivo.html sacunapolo:/home/sacunapolo/ ssh sacunapolo sudo chown www-data:www-data /home/sacunapolo/archivo.html

❌ Error: Problemas de conexión SSH

Solución: Primero prueba la conexión SSH básica:

ssh sacunapolo

Si SSH funciona, SCP también debería funcionar.

❌ Error: Carpetas con espacios en nombres

Solución: Usar comillas o escape con barra invertida:

# Con comillas scp "mi archivo.txt" sacunapolo:/home/ # Con escape scp mi\ archivo.txt sacunapolo:/home/

Consejo Final para Transferencias Frecuentes

Si necesitas transferir archivos con frecuencia o mantener sincronizadas carpetas, usa rsync en lugar de scp:

rsync -avz *.html sacunapolo:/home/sacunapolo/

Ventajas de rsync: Solo transfiere cambios, mantiene permisos y es más rápido en transferencias posteriores.
